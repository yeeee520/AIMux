# AIMux 使用文档

## 简介

**AIMux** 是一款 Windows 桌面端 AI 编码代理工具，用于为 AI 编码工具（Codex 等）配置自定义 API 路由。你可以将不同模型指向不同的 API Provider。

---

## 安装

双击 `AIMux.exe` 即可运行。

> 首次运行会在 `%APPDATA%\AIMux\` 自动生成配置文件和图标，不会污染 exe 所在目录。

---

## 界面概览

4 个功能标签页 + 底部固定栏：

| 标签 | 功能 |
|------|------|
| 控制台 | 启停代理、查看状态、统计信息、使用指引 |
| API Keys | 管理 API Provider |
| 路由规则 | 配置模型 → Provider 的路由规则 |
| 请求日志 | 查看实时 API 请求日志 |

底部固定栏：**快速接入** — 一键配置 Codex，无需手动编辑文件。

---

## 使用步骤（以 DeepSeek 为例）

### 第一步：添加 Provider

1. 切换到 **「API Keys」** 标签页
2. 点击 **「+ 添加 Provider」**
3. 填写：

| 字段 | 值 |
|------|-----|
| 名称 | `DeepSeek` |
| Base URL | `https://api.deepseek.com` |
| API Key | 你的 DeepSeek API Key |
| 模型列表 | `deepseekV4pro` |

4. 点击保存

### 第二步：一键配置 Codex

在窗口底部的快速接入栏：

- 工具选 **Codex**
- 模型名填 `deepseekV4pro`
- Provider 选 **DeepSeek**
- 点 **⚡ 一键配置**

> 这会自动修改 `~/.codex/config.toml`，将 Codex 的 API 请求指向本代理。

### 第三步：启动代理

1. 切到 **「控制台」** 标签
2. 确认端口（默认 3456）
3. 点 **「▶ 启动代理」**

状态灯变绿即可。然后重启 Codex 就能用了。

---

## 系统托盘

关闭窗口时程序不会退出，而是最小化到系统托盘：

- **双击托盘图标**：显示/隐藏窗口
- **右键菜单**：显示窗口、启停代理、退出

---

## API 端点

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | `/v1/chat/completions` | OpenAI 兼容格式 |
| POST | `/v1/responses` | Codex 专用端点 |
| GET | `/v1/models` | 模型列表 |
| GET | `/api/config` | 获取配置 |
| PUT | `/api/config` | 更新配置 |

---

## 配置文件

配置文件保存在 `%APPDATA%\AIMux\config.json`，结构如下：

```json
{
  "proxyPort": 3456,
  "providers": [{ "id": "...", "name": "...", "baseUrl": "...", "apiKey": "...", "models": "..." }],
  "rules": [{ "id": "...", "name": "...", "modelPattern": "gpt-4o*", "providerId": "...", "modelOverride": "" }]
}
```

## 常见问题

**Q: Codex 报 404 Not Found？**
确保已勾选 Codex 使用 `wire_api = "responses"`，代理服务器已支持该端点。

**Q: 端口被占用？**
在控制台修改端口号后重启代理。

**Q: 如何查看请求是否成功？**
切换到「请求日志」标签页，实时显示每个请求的状态码。
